## Audited but not planned (still open findings)

These came out of the same audit and were **not** selected for plans. They are listed so
the next session does not have to rediscover them, roughly by leverage:

1. **No CI** — `npm test` / `npm run lint` / `npx tsc --noEmit` / `npm run check:format`
   all exist but run nowhere automatically; there is no `.github/` at all. Cheapest
   high-leverage gap in the repo (effort S).
2. **Client-side scoring** (security, effort L) — points are computed on-device and
   `team_points` is written straight to `team_answers`
   (`answerStorage.ts:14-22`, `offlineOutbox.ts:261-269`), and the correct-answer flag
   is shipped to the client. `utils/Supabase.ts:5` carries the matching
   `// TODO enable proper authentication in supabase`. Needs a design spike coordinated
   with `campus-rallye-admin` (RPC/trigger that recomputes points server-side).
   Note: plan 008 consolidates the client's `pointsAwarded` argument into one hook,
   which is the seam this work would remove.
3. **Photo upload pipeline** (perf + security, effort M) — full-resolution photos are
   base64-decoded into a `Uint8Array` before upload (`answerStorage.ts:52-58`,
   `UploadPhotoQuestion.tsx:293` takes the picture with no `quality`/resize), and object
   names are guessable (`${teamId}_${questionId}.jpg`, `answerStorage.ts:62`) with no
   size cap.
4. **`setPlayTime` swallows its error** (`teamStorage.ts:27-33`) — no error check, no
   offline retry, so a team's `play_time` can be lost silently. Plan 002 characterizes
   this behavior in a test; fixing it is a separate change.
5. **`@legendapp/state` pinned to a beta via caret** (`package.json:18`,
   `^3.0.0-beta.42`) — beta bumps can float into the offline/answer state path.
6. **Outbox redaction gap + unbounded retries** (`offlineOutbox.ts:53-66`) — the
   redaction pattern misses Postgres's `Key (field)=(value)` detail format; low
   sensitivity (quiz answers in app-private AsyncStorage).
7. **GPS/sensor duty cycle during geocaching navigation**
   (`GeocachingQuestion.tsx:219-224,282`) — `BestForNavigation` + 1 m/1 s + DeviceMotion
   at 20 fps alongside WebGL rendering. Needs on-device measurement before changing.
8. **Styling-system drift** — `GlobalStyles` / `AppStyles`+`makeStyles` / `buttonStyles` /
   `Typography` / `spacing` coexist, plus hardcoded `rgba(226, 0, 26, …)` literals
   (e.g. `scoreboard.tsx:175-180`) that bypass the theme.

### Direction options (for the maintainer, not defects)

- **Offline asymmetry**: outbound answers are hardened (outbox with retry/backoff), but
  questions load live-only — a team in a campus dead zone can queue answers yet cannot
  load the next question. Caching the joined rallye's questions would make a started
  rallye fully playable offline. Effort L; open question is cache invalidation when the
  admin edits questions mid-rallye.
- **Typed question-type registry**: adding a question type today means editing
  `types/rallye.ts`, the `Record<string, any>` map in `question-renderer.tsx:25-32`, and
  i18n keys — plus the sibling admin repo; mismatches fail silently to the skip screen.
  A registry derived from the `QuestionType` union would make `tsc` catch unmapped types.
  Effort M.
